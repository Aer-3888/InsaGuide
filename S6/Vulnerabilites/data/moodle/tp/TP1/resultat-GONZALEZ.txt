### Analyse du contenu du fichier

Un aspect intéressant de l'analyse de la page web est de trouver les balises HTML les plus fréquemment employées dans la page. On peut avoir un aperçu avec
```
cat pixgame.html.prep | sed 's/</ </g' | tr ' ' '\n' | grep '^<[^!]'
```

On obtient un premier découpage assez sale de la structure des balises qui nous indique une multitude de balises de script. Le seul inconvénient est que l'on attrape aussi dans ce découpage des comparaisons en javascript (des chaînes comme "a<b" qui deviennent "a <b" puis "a\n<b" alors que "<b" n'est pas la balise HTML ici).

Je filtre une fois de plus pour enlever les balises fermantes et j'ajoute un `uniq -nr` pour trier et compter.

```
cat pixgame.html.prep | grep -v '^\s*$' | sed 's/</ </g' | sed 's/>/> /g' | tr ' ' '\n' | grep '^<[^!/]' | cut -b '2-' | sed 's/>//g' | sort | uniq -c | sort -nr
```

### Authentifiants WP-admin

Tout semble indiquer que le site tourne sur la plateforme "WordPress" (présence du sous-dossier wp-contents, wp-admin)

Avec
```grep "admin" pixgame.html.prep -n```
on trouve immédiatement des endroits intéressants, y compris des scripts qui effectuent des appels via l'interface administrateur de wordpress. Pour ce faire, les scripts doivent très certainement posséder des identifiants qui nous permettent peut-être de pénétrer ce panel administrateur.

Vers la ligne 127, la définition de la structure `twdGlobal` de l'objet javascript `window` interpelle puisqu'elle contient :
 - L'URL vers le panel administrateur du moteur wordpress
 - Le nonce(!!!) utilisé pour l'authentification auprès d'une API REST
 - L'URL de l'API REST du backend wordpress
 - un format de permalink vers les posts du site

La présence du nonce, si ce dernier permet effectivement de s'authentifier, est *gravissime*. Un nonce est par définition utilisé dans une seule session de protocole et aléatoire. Même si ce dernier est remplacé au cours de l'exécution des scripts sur la page à chaque rafraichissement, il est très inquiétant qu'un utilisateur qui se contente de naviguer sur le site puisse avoir accès aussi simplement à une valeur censée rester privée.

L'ouverture de l'URL de l'API REST sur un navigateur révèle d'autres données intéressantes à exploiter, mais cela sort du contexte de l'analyse pure du fichier.
